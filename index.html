A bit of history

A few days ago I got an e-mail from my daughter's school saying they published new photos on their Flickr site. Excited I opened the page and started to go through and quickly realized that it's just a nightmare. I have to find my daughter on small preview images full of other kids and then do a lot of other clicks to download a single photo. It will be much better just to have full images and flip through them in a desktop application or just start a slide show. Besides it will be great to have all these photos because it's a perfect snapshot of her life, which will be priceless in 10 or 20 years. So, I decided to download all photos in a maximum available size and found that, well... it's a lot of work! Go image by image, for each select a size, open it in a page and then do "Save as..." can easily take a half of hour or a hour for an album with 100 or 200 photos. Also it's really annoying and boring.

After a quick Google search I was very exited because there are many tools for that task listed right on Flickr: http://www.flickr.com/services/apps/tags/download. The only problem is that they don't work one way or another. Some of them are not automatic, some just through errors, some allow downloads only of low resolution photos, some start to ask for money on the very last step. After few hours of trying I still didn't have all images downloaded and finally gave up.

This project was born just out of dispair. The idea is very simple: create a command line application, which loads a simple configuration file with a list of albums to download. On each run it scans albums for available images, compares them with files on disk and if there are differences then it downloads missing images. It helps to solve problems with some Flickr servers, which return errors. Just run this application several times and finally it will get everything. Because it's a simple executable, it's possible to run it on a schedule to monitor sites. The implementation is in .NET, so it can be used on both Windows or Unix (using Mono).
